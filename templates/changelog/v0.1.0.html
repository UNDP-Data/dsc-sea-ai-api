This release streamlines graph extraction process, extends parameters for /graph endpoint and adds basic API tests.


What's New
  • /graph endpoint now uses vector search which enables querying it in various languages. For example, "energia solare" will return a graph matching "solar energy". Graph data are still in English only.
  • Optional hops parameter has been added to control graph size. Setting this parameter to zero will result in only a central node being returned, while one and two will produce 1- and 2-hop neighbourhoods respectively. That is, it will produce nodes that are no more than hops edges away from the central node.
  • weight property on nodes now corresponds to cosine similarity between a query and a node. It is a floating-point number in the [0.0, 1.0] range with higher values indicating closer similarity.
  • Nodes have gained two new properties: neighbourhood, which is a non-negative integer corresponding to the distance from a central node to this node, i.e., 0 for the central node, 1 for nodes that are 1-hop away and so on, and colour, which is a hex colour value for the node. Nodes are coloured along shortest paths from the central node such that the nodes in the same path share the same colour but use a different shade of it.
  • Basic API tests are now included in the codebase.

Bug Fixes
  • Previously, edges could contain subject or object names that were not present among nodes. This is no longer the case and graph edges and nodes are consistent.
